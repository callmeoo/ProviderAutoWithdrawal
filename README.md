# 中间服务商提现优化 · 需求原型

> 一句话：把**中间服务商**「初始化款 / 服务费」的提现，从人工逐笔审批改造为**系统自动发起 → 自动对备注 → 中控自动通过**的直通流程；失败时企微告警通知责任人。普通买家提现流程保持不变。

一份**可交互的需求文档原型**（纯静态 HTML，无需构建），供产品评审与研发对齐使用。

## 快速预览

- **直接打开**：用浏览器打开 `index.html`（需联网，Vue / Tailwind / marked 走 CDN）。
- **本地服务器**（推荐，iframe 加载更稳）：
  ```bash
  python3 -m http.server 8000
  # 浏览器访问 http://localhost:8000/index.html
  ```

## 需求核心

- **入口控制**：中间服务商点击「可用余额提现」仅弹自动提现提示，不进入提现申请流程；不限制保证金入口。
- **自动提现**：单结订单完成清算并提车后自动发起；月结于次月 10 日 10:00 统一发起；超 10 台拆分多笔；备注自动汇总车牌。
- **售后抵扣**：已提车 / 已回收资料后发生退车，服务费按成交时间从早到晚依次抵扣，抵扣后仍有余额再自动提现。
- **审批流**：系统自动发起的提现，中控中心节点自动通过；后续仍走财务 / 经管 / 出纳节点。
- **失败处理**：自动提现失败企微告警，不自动重试、转人工处理。
- **上线初始化**：上线前完成服务商范围、结算方式、历史服务费状态初始化，并输出核对清单供业务核对。

## 技术栈

纯静态 HTML + Vue 3（global build）+ Tailwind CDN + marked.js + lucide，全部走 CDN，**无构建步骤**。
